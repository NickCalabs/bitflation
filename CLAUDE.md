# Bitflation

Bitcoin inflation-adjusted price visualizer. Shows BTC priced in "real" dollars using CPI, M2, Gold, DXY, and the Bitflation Index (BFI).

## Status

- **V1:** CPI-adjusted BTC chart
- **V2:** M2, Gold, DXY metrics + log scale
- **V3:** Real-time data (FRED API), landing page explainer, shareable URLs, calculator — DONE
- **V3.5:** Bitflation Index (BFI) composite metric, multi-metric overlay, comparison assets, Real Price view, click-drag zoom, event annotations, OG image — DONE
- **V3 backlog:** multi-asset comparison polish — NOT STARTED

## Tech Stack

- **React 19** + **TypeScript 5.9** (strict mode, no emit)
- **Vite 7** (dev server, bundler)
- **Recharts 3** (charting)
- **date-fns 4** (date utilities — currently light usage)
- **CSS Modules** (scoped component styles, no CSS-in-JS)
- No state management library — plain `useState` + `useMemo`

## Deployment

- **Hosting:** Cloudflare Pages (manual deploy via `npx wrangler pages deploy dist/ --project-name bitflation`)
- **OG image:** Cloudflare Pages Function at `functions/og.png.tsx` — server-side rendered PNG using `@cloudflare/pages-plugin-vercel-og`
- **Build + deploy flow:** `npm run build && npx wrangler pages deploy dist/ --project-name bitflation`
- **VITE_ env vars are build-time only** — they get baked into the JS bundle by Vite. They must be present in `.env` at build time. Cloudflare Pages "secrets" are runtime-only and do NOT work for VITE_ vars. If building on Cloudflare's CI, set them as plaintext environment variables in the dashboard.

## Data Pipeline (App.tsx)

All state and data flow lives in `App.tsx`. No context providers, no reducers.

```
Static JSON + live fetches (CoinGecko/Blockchain BTC, FRED DXY + M2 + SP500)
  → dataReady gate (skeleton until ALL fetches resolve/reject)
  → stitchPrices / stitchDeflators (merge, live wins on overlap)
  → interpolateMonthlyToDaily (CPI, M2, Gold → Map<date, value>)
  → computeBitflationIndex(dailyCpi, dailyM2, anchorYear) → BFI deflator
  → branch on metric:
      BFI/CPI/M2/DXY → adjustPrices(prices, deflatorMap, anchorYear)
      GOLD            → convertToGold(prices, goldMap)
  → filterByTimeframe(data, '1Y'|'5Y'|'ALL')
  → render chart + calculator + explainer
```

Key types flowing through the pipeline:
- `PricePoint` → raw `{ date, price }`
- `AdjustedPricePoint` → `{ date, nominalPrice, adjustedPrice }` (BFI/CPI/M2/DXY)
- `GoldPricePoint` → `{ date, nominalPrice, goldOunces, goldPriceUsd }` (Gold)
- `MultiMetricPoint` → `{ date, nominalPrice, bfiAdjusted?, cpiAdjusted?, m2Adjusted?, dxyAdjusted? }`
- `LiveDataStatus` → `'all' | 'partial' | 'none'`
- `InflationMetric` → `'BFI' | 'CPI' | 'M2' | 'GOLD' | 'DXY'`
- `DeflatorMetric` → `'BFI' | 'CPI' | 'M2' | 'DXY'` (subset that uses adjustPrices)

## Critical Pattern: dataReady Gate

**Problem solved:** Without gating, React renders immediately with static data (stale BTC price from JSON), then re-renders when live data arrives — causing a visible price jump.

**Solution:** `dataReady` state starts `false`. Set to `true` ONLY in the `.then()` of `Promise.all` (both success and rejection handlers). Until `dataReady` is true:
- HeroPrice receives `null` → shows pulsing skeleton bars
- PriceChart, Calculator, Explainer are **unmounted** (not rendered to DOM at all)
- All `setLive*` state updates + `setDataReady(true)` happen in the same callback → React batches into ONE re-render with final data

**Never break this pattern.** If you add new data sources, they must complete before `dataReady` flips.

## Bitflation Index (BFI)

The default metric. A 50/50 blend of CPI and M2 growth, normalized to the anchor year.

**Math:** `bfi(date) = 0.5 × (cpi/cpiAnchor) + 0.5 × (m2/m2Anchor)`

**Key difference from other deflators:** BFI depends on `anchorYear` (to compute CPI/M2 anchor averages), so `dailyBfi` is recomputed via `useMemo` when anchor changes. Other deflators (CPI, M2, DXY) are raw values independent of anchor — `adjustPrices()` handles the anchor math.

**Why it works with adjustPrices():** The BFI values are ~1.0 for anchor-year dates. When `adjustPrices()` computes its anchor average, it gets ~1.0 and divides by it — effectively a no-op. The real normalization is already in `computeBitflationIndex()`. This is NOT double-application.

**Color:** `#e4e4e7` (white/zinc) — stands out as "primary" on dark background.

## Live Data Sources & Fallbacks

| Data | Primary | Fallback | Key needed |
|------|---------|----------|------------|
| BTC price (client) | CoinGecko (365d daily) | Blockchain.com ticker (today only) | `VITE_COINGECKO_API_KEY` (demo, free) |
| BTC price (OG image) | CoinGecko (spot) | Blockchain.com ticker | None (no key needed for spot) |
| DXY | FRED API | Static JSON | `VITE_FRED_API_KEY` (free) |
| M2 | FRED API | Static JSON | `VITE_FRED_API_KEY` (free) |
| S&P 500 | FRED API | Static JSON | `VITE_FRED_API_KEY` (free) |
| CPI | Static only | — | — |
| Gold | Static only | — | FRED removed daily LBMA series |

**CoinGecko demo API rate-limits aggressively** (~10-30 req/min). During testing, expect 429s. The Blockchain.com fallback ensures at least today's spot price is always available.

**OG image debug headers:** `X-Price-Source: coingecko|blockchain|none` and `X-Debug-Error` on the `/og.png` response. Check with `curl -sI https://bitflation.io/og.png | grep X-`.

## Key Patterns

**Deflator math:** `adjustedPrice = nominalPrice × (deflatorAnchor / deflatorCurrent)` where anchor is the average deflator value for the selected year.

**Monthly → daily interpolation:** Linear interpolation between monthly data points, then flat-hold for 90 days past last data point. Used for CPI, M2, and Gold. DXY is already daily (forward-filled for weekends/holidays).

**Map-based lookups:** All deflator data stored as `Map<string, number>` keyed by `YYYY-MM-DD` for O(1) date lookups.

**DEFLATOR_LOOKUP and DEFLATOR_MAP_MEMO:** Two nearly-identical `useMemo` objects mapping `DeflatorMetric → Map<string, number>`. `DEFLATOR_LOOKUP` is used for `processedData` (single metric) and `comparisonData`. `DEFLATOR_MAP_MEMO` is used for `multiMetricData` (multi-metric overlay). These could potentially be unified.

**Graceful degradation:** If API keys are missing or requests fail, app works from static data. Footer badge shows three states: green (all live), yellow (partial), red (static only).

**Shareable URLs:** Controls state syncs to URL params (`metric`, `anchor`, `tf`, `log`) via `history.replaceState`. Only non-default values are written to keep URLs clean. Default metric is now `BFI` (was CPI).

**Log scale safety:** Recharts log scale breaks on values ≤ 0, so `PriceChart` filters those out before rendering when `logScale` is true. Domain minimum is set to `1`.

**HeroPrice skeleton:** When `latestPoint` is null, renders pulsing gray bars (`@keyframes pulse`) matching the layout of real content. When data arrives, fades in with `animation: fadeIn 200ms ease-out`.

## OG Image (functions/og.png.tsx)

Server-side rendered PNG via Cloudflare Pages Function. Uses hardcoded CPI/M2 constants for BFI computation (can't import client-side data). **These constants must be updated quarterly:**

```typescript
const CPI_ANCHOR_2015 = 237.017;
const CPI_CURRENT = 325.252;      // Update with latest CPI-U
const M2_ANCHOR_2015 = 12055.6;   // 2015 average M2 (billions)
const M2_CURRENT = 22411;         // Update with latest M2 value
```

If both price sources fail, renders a brand-only image (logo + tagline, no dollar amounts) instead of showing a wrong number.

## Gotchas & Lessons Learned

1. **Static data goes stale.** BTC data was stuck at Jan 2025 ($102k) while live price was $68k. Run `prepare-btc-data.mjs` periodically. The script now uses dynamic end date.
2. **CoinGecko rate-limits the demo key aggressively.** Always have a fallback (Blockchain.com ticker works without a key).
3. **Cloudflare Pages secrets ≠ build-time env vars.** `VITE_` prefixed vars must be plaintext environment variables in the build settings, not "secrets" (which are runtime-only for Workers).
4. **React renders static data instantly before fetches complete.** The `dataReady` gate prevents this — never show data-dependent UI until the fetch round-trip is done.
5. **OG image CPI/M2 constants are hardcoded.** The Cloudflare Function can't import client-side modules. Update manually when refreshing other data.
6. **METRIC_COLORS and METRIC_KEYS are duplicated** in PriceChart.tsx and CustomTooltip.tsx. If you add a metric, update both.
